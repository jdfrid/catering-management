מדריך משתמש

מדריך שלב-אחר-שלב לניהול קייטרינג מקצה לקצה: לקוחות, אירועים, תפריט, הצעות מחיר, רכש, משימות מטבח, כספים וייבוא נתונים.

גרסת MVP · עברית (RTL) · catering-management

1. התחלה — כניסה למערכת

  1. פתחו את כתובת המערכת בדפדפן (מקומית: http://localhost:3000, בפרודקשן: כתובת Render).
  2. אין עדיין מסך התחברות — כל המשתמשים רואים את אותם מסכים.
  3. אם מופיעה שגיאת «לא ניתן לטעון את העמוד»: ודאו ש־PostgreSQL פועל (npm run db:up), הריצו npx prisma migrate deploy, ובדקו ש־DATABASE_URL מוגדר בקובץ .env.

3. זרימת עבודה מומלצת

לקוח → אירוע → תפריט + מחירון → הצעת מחיר → אישור → מטבח → רכש → תשלום
  1. יצירת לקוח (שם + טלפון חובה).
  2. פתיחת אירוע — תאריך, מספר מוזמנים, רמת שירות, כתובת.
  3. הגדרת תפריט: מנות, חומרי גלם, שורות מתכון, מחירון ספקים.
  4. יצירת הצעת מחיר לאירוע + הוספת שורות (מנות וכמויות).
  5. שמירה עם סטטוס «אושרה» — מעדכן את האירוע ויוצר משימות מטבח אוטומטית.
  6. רכש: הזמנות לספקים (קישור לאירוע אופציונלי).
  7. מטבח: ביצוע משימות ועדכון סטטוס.
  8. כספים: רישום תשלום מהלקוח.
טיפ: ניתן לייבא נתונים מראש מ־CSV (סעיף 12) במקום הזנה ידנית.

4. לקוחות

  1. לחצו «לקוחות» בסרגל → «לקוח חדש».
  2. מלאו שם, טלפון (חובה), סוג לקוח (פרטי / עסקי / מוסדי), אימייל, כתובת, ח.פ, הערות.
  3. שמרו — תועברו לכרטיס הלקוח לעריכה, צפייה באירועים והצעות.

5. אירועים

  1. «אירועים» → «אירוע חדש» → בחרו לקוח.
  2. מלאו סוג אירוע, תאריך ושעה, מספר מוזמנים, רמת שירות (שירות מלא / הגשה / משלוח / איסוף עצמי).
  3. הוסיפו שם אירוע, כתובת, שעת אספקה, סוג תפריט והערות תפעול.
  4. בעמוד האירוע: הצעות מחיר, משימות מטבח (לאחר אישור הצעה), קישורים לרכש וכספים.

6. תפריט ועץ מוצר

6.1 חומרי גלם

  1. «תפריט» → «חומרי גלם» → «חומר חדש».
  2. שם, סוג (יבש / קירור / בשר…), יחידה: מנה, מגש, ק״ג, גרם, ליטר, יחידה.
  3. ספק מועדף (אופציונלי) — משמש בחישוב עלות.

6.2 מנות ומתכון

  1. «תפריט» → «מנה חדשה» — שם, קטגוריה, יחידת מכירה, כמות בסיס למתכון.
  2. בעריכת מנה: «עץ מוצר — פירוק למתכון» — הוסיפו חומרי גלם, כמות, יחידה (למשל גרם), אחוז פחת.
  3. קטע «עלות משוערת לפי מחירון ספקים» — חישוב אוטומטי (סעיף 8).

7. הצעות מחיר

  1. «הצעות» → «הצעה חדשה» — בחרו אירוע (או מעמוד האירוע: «הצעה חדשה לאירוע»).
  2. הוסיפו שורות: מנה, כמות, מחיר ליחידה — הסיכום (לפני מע״מ, מע״מ, סה״כ) מתעדכן אוטומטically.
  3. «פרטי הצעה»: הנחה, עלות פנימית, תנאי תשלום, הערות, תוקף.
  4. לאישור: שנה סטטוס ל«אושרה» ושמרו — האירוע יסומן מאושר, וייווצרו משימות מטבח (סעיף 10).
  5. הצעה מאושרת ננעלת לעריכה (MVP).

בעמוד ההצעה: עמודת «הערכת עלות» לכל שורה + סיכום «סה״כ הערכת עלות (מחירון)» בסרגל.

8. חישוב עלות אוטומטי

המערכת מחשבת עלות מתכון לפי:

  • מתכון פעיל של המנה (כמויות + פחת).
  • מחירון ספקים — יחידת המחיר חייבת להתאים ליחידה בשורת המתכון.
  • ספק מועדף לחומר (אם קיים מחיר תקף), אחרת המחיר הזול ביותר.
  • טווח תאריכי תוקף במחירון (אם הוגדר).

איפה רואים: בעמוד עריכת מנה; בהצעת מחיר (עמודת הערכת עלות).

מחירון: רכש → מחירון ספקים.

9. רכש

  1. ספקים — רשימה, יצירה, עריכה; בכרטיס ספק גם טבלת מחירון.
  2. מחירון ספקים — מחיר לפי ספק + חומר גלם + יחידת מחיר (עדכון = upsert).
  3. הזמנות רכש — טיוטה, שורות (שם חומר, כמות, יחידה, מחיר משוער), סטטוס; קישור לאירוע אופציונלי.

10. משימות מטבח

יצירה אוטומטית: כשהצעת מחיר נשמרת כ«אושרה» ויש בה שורות מנות — נוצרת משימה לכל מנה (כמות מסוכמת), תאריך עבודה = תאריך האירוע, מחלקה «הכנה מוקדמת». ההערה מתחילה ב־[אוטו-הזמנה].

  1. «מטבח» — כרטיסי משימות מהיום ואילך; עדכון סטטוס מהיר.
  2. «משימה חדשה» — יצירה ידנית (אירוע, מנה, מחלקה, תאריך).
  3. בעמוד האירוע — רשימת משימות מקושרות.

11. כספים

  1. «כספים» → רישום תשלום: לקוח, סכום, תאריך, אמצעי תשלום.
  2. קישור לאירוע — מעמוד האירוע («פעולות מהירות») ממלא מראש.

12. ייבוא נתונים מ־CSV (אקסל)

  1. «ייבוא» בסרגל העליון.
  2. באקסל: קובץ → שמירה בשם → CSV UTF-8 (מומלץ).
  3. בחרו סוג רשימה (לקוחות, ספקים, מנות, מחירון, שורות מתכון, אירועים…).
  4. העלו קובץ — שורה ראשונה = כותרות (עברית או אנגלית).

סדר מומלץ:

  • לקוחות → אנשי קשר (אופציונלי) → אירועים
  • ספקים → חומרי גלם → מנות → מחירון → שורות מתכון

שמות (ספק, חומר, מנה) חייבים להתאים בדיוק לשם במערכת.
